Godot - 节点

发表于 2026-03-17 更新于 2026-03-18 分类于 技术 ， Godot

前言

在 Godot 中开发游戏时，节点（Node）是构建游戏的核心元素，它们通过层级化的场景树（Scene Tree）组织起来，实现游戏逻辑、渲染、物理等所有功能。

1. 基础节点（Core Nodes）

作用：所有节点的基类，提供通用功能（如名称、父子关系、信号等）

2. 2D 渲染节点（2D Rendering Nodes）

作用：处理 2D 图像渲染、动画和视觉效果

3. 物理与碰撞节点（Physics & Collision Nodes）

作用：处理 2D 物理模拟和碰撞检测

4. 控制与 UI 节点（Control & UI Nodes）

作用：构建用户界面（HUD、菜单）和交互元素

5. 逻辑与工具节点（Logic & Utility Nodes）

作用：实现游戏逻辑、状态管理和工具功能

6. 场景与实例化节点（Scene & Instantiation Nodes）

作用：动态加载和管理场景（肉鸽游戏的核心机制）

  • 向量指有 大小、有方向的量

模相等、方向相同的向量称为 相等向量 ，模相反、方向相反的向量称为 相反向量

在代码中，向量通过用数对来表示，注意！！！Godot中Y轴正方向是向下的！ 向量(x,y)可以表示以原点为起点，(x,y)为终点的向量

根据平面向量基本定理可 把向量分解为沿x、y轴的分量

  • 向量的运算

向量(a,b)和向量(c,d)相加为(a+c,b+d)

向量(a,b)和向量(c,d)相减为(a-c,b-d)

因为这和向量分解后再相加相减一样的

二维向量的数乘：向量(a,b)乘以 x 得(ax,bx)，相当于把这个向量等比扩大x倍或缩小x倍

二维向量的数量积（点乘）：向量(a,b)和向量(c,d)相乘为 ac+bd，是一个数量！（有大小无方向的量，又称标量），攻就是一个物体所受力与其位移的数量积
