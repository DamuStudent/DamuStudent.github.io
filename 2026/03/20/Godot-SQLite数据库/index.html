Godot - SQLite数据库

发表于 2026-03-20 更新于 2026-04-04 分类于 Godot ， Godot学习笔记

前言

为了更好的为我的毕设项目服务，我发现普通的 resource类 无法满足我对数据存储的需要，我开始寻找新的办法，我尝试使用 MySQL ，我发现 MySQL 更适合使用在大型的复杂的联网的多人的商业项目中，和我并不适配。

最后，我找到了SQLite，这是一个 Godot 中的插件数据库，听上去非常适合我的小项目

参考教程：godot4.X中的SQLite使用教程_哔哩哔哩_bilibili

⚡ 更正：SQLite 是一个底层数据库引擎，而我们真正用到的 godot-sqlite 是Godot端的封装接口，在对 ai 提问时请描述清楚，这有助于帮助你更高效的解决问题，在本文档中，将 SQLite 代指 godot-sqlite

注意事项

  • ⚡请注意，在 SQLite Brower 中对表的任何操作，都一定要保存！不论是修改表的字段，删除表或者字段，新建表，不然 GDScript 中的代码不会起任何作用！且不会报错！

  • ⚡请注意，对于主键，假如你插入三条记录，id分别为 1,2,3，删除id为2的记录，再插入数据，那么id为1,3,4，主键一定唯一！

  • ⚡请注意，如果你往一个 数字类型 的字段插入了字符，它的值会是 0

  • ⚡请注意，在SQLite为我们封装的常用方法中，条件表达式一定是 列名 = '值'

  • ⚡请注意，SQLite 返回的大多数查询结果为 数组！

  • ⚡请注意，SQLite 中，是要区分 空格 的！

表格数据类型：BLOB

我将逐条解释上述代码的含义

  • 图片上传数据库

var image := preload("res://assents/图片/家传粗糙大宝剑.png") 预加载本地路径的图片资源，**preload**返回的是一个 Texture2D 对象（Godot 中图片资源的通用类型）

var pba = image.get_image().save_png_to_buffer() image.get_image()提取原始 Image 对象，是一个未压缩的RGBA数组 ，而**save_png_to_buffer()** 则将 Image 对象编码为 PNG 格式的二进制数据（PackedByteArray），一个包含PNG文件的完整二进制数据，这样才能直接存储到 BLOB 字段中

  • 图片从数据库加载到本地

var image = Image.new() image.load_png_from_buffer(i.Img) 这两句的意思是，image为Image实例，将查询结果中的 二进制PNG数据 解析为 Image 对象

var texture = ImageTexture.create_from_image(image) 将 Image对象 转成 Texture2D 对象，供下面的TextureRect节点 显示

  • 综上所述，不论是上传还是下载，都需要转两次格式

上传：Texture2D => Image对象（RGBA数组） => PNG二进制数据 => 存储到 BLOB 字段中

下载： PNG二进制数据 => Image对象（RGBA数组） => Texture2D => 显示到 UI节点 中
