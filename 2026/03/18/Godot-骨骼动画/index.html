Godot - 骨骼动画

发表于 2026-03-18 更新于 2026-03-30 分类于 Godot ， Godot学习笔记

前言

ps. 一些小沐的感慨

针对2d游戏，比较主流的动画渲染有诸如 帧动画、骨骼动画、三渲二等…我只接触过帧动画，借着以前的一些绘画热情和好用的 AI，也零零散散画了许多，但是帧动画的限制很大，每一个动作最少也要 画 4 帧 才看得过去，每为 Player 添加一个新的动作或者新的武器，你要画的素材也会指数上涨，再加上我现在的游戏构思，我将开始学习骨骼动画

二、开始学习

新建 player 场景，并添加 CharacterBody2d 节点，为父节点添加 Node2d 节点作为视觉容器，为 Node2d 节点添加 Skeleton2D 节点（骨架），为骨架添加第一根骨头节点 Bone2D 胯部，为胯部添加 3 个骨头，分别是 躯干、左腿、右腿，选中 躯干 节点往上移动，左腿 左移，右腿 右移

  • 放入身体图片，将 精灵 放于 身体 节点下

⚡ ⚡ ⚡ 将 精灵 图片的 position 属性归零！通过偏移量 offset 手动调整至合适位置

  • 左腿 以及 左脚

将左腿节点移动至 左腿旋转的位置中心，为左腿添加子骨头节点 左脚 ，把 左腿 精灵图放入 左腿节点下，和身体一个步骤，position 归零，通过偏移将位置调整合适，当你准备调整左脚时，请把左脚的自动设置长度( Auto Calculate) 关掉，通过手动设置长度和角度设置左脚的图片至合适位置，注意骨头伸出来的节点是你下一个骨头的移动的中心

  • 其他部位

跟随教程继续…

我想实现的和教程有些许出入，教程中腿是由两个骨头制作的，而我的人物因为腿很短所以只用了一个骨头

其他部位和教程一样，新建过程也一样

  • 跟随 marker2D 节点移动

当你把各骨头创建好了（创建一个也可以），你会好奇怎么使得骨骼动起来，当然可以通过旋转角度使得骨骼根据中心点转动作为调试。

实际上，我们可以通过在 Skeleton2D 的检查器中设置 Modifications 来控制骨头的移动，一个游戏人物，假如它的四肢都是由两根骨头组成的，我们在 Modifications 中需要创建四个元素，且资源都选择新建 SkeletonModification2DTwoBoneIK ，就如它的名字一样，它针对 有两根骨头 且 希望他们通过数学运算后能指向某一点时使用，它需要你有一个 marker2D 节点作为 目标点，以及两根连在一起的 骨头 ，最后别忘了点击 启用

那么，对于头这种只有一根骨头的，我们将选择 SkeletonModification2DLookAt ，它需要你提供一个 Marker2D 节点，和一个 Bone2D

不推荐通过调整 z轴索引 的方式排列顺序，这样在后续与其他怪物和场景交互时会出现问题

⚡节点越靠下，显示层级越高！

  • 胯部目标

通过为胯部骨头设置目标点，可实现更复杂更精细的动画，比如移动时身体微微摆动

  • 拿武器的思路

通过额外绘制 手部抓握 素材 + 武器 素材，所有的武器和防具将有唯一的 id 且只要 武器种类 不一样，id前缀不一样，我将把这写入项目文档

  • 拿双手武器的思路

详见教程，大概是固定 左右手目标点位

  • 动画

通过 Animationplayer 制作动画，为每个目标点位的位置打上关键帧

⚡注意，关键帧不能打少，会显得很人机，如果一个动作只打两个关键帧，那么目标点位只会在两点直接移动，想想那个恐怖的画面吧，至少三个关键帧会使你的动作变得流畅

⚡⚡注意，如果你为你的骨骼旋转设置了范围，在设计动画时，就算你把目标点位拖出范围也不会让骨骼超出，但是千万不能这么干！因为你打的是 目标点位 的关键帧，如果你的目标点位离范围较远，你会发现动作会跳的很快！！！虽然你的骨骼无法超出范围，但是 Godot 识别的终点仍然是你的目标点位，骨骼会在很短时间内移动到范围极限然后停下！！！

三、和视频不一样的需求

  • 对于我开发的游戏，我希望的是玩家可以通过鼠标确定武器和头朝向的方向，通过键盘 WASD 来控制移动，所以和视频中的最后样板产生了冲突

情况1：当我看向右边但是往左走时，脚的动画应该是 向左边退步

情况2：当我看向右边且往右边走时，脚的动画应该是 向右边移动

当我仔细想了一下，我发现：我的游戏人物的腿部很简单，只有一根骨头构成，所以，他奶奶的她往前走和往后走的区别不大！所以移动只需要一个动作就够了，接下来是当我看向人物右边时人物应该水平翻转，我的鼠标还要绑定 手部，头部目标点
